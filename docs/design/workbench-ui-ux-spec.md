# Gabaki / Sabaki 工作台视觉复现与前端实现规格

> 目标：以前端实现能够高保真复现 `workbench-ref-pics` 当前三张参考图为准，定义 Sabaki 魔改工作台的布局、视觉层级、模式差异、交互状态机和验收标准。本文涵盖视觉复现与关键交互行为，但不作为完整产品功能蓝图。

## 0. 参考图复现目标

本文的视觉真源是 `docs/design/workbench-ref-pics/` 下当前三张 PNG：

- `ChatGPT Image 2026年4月30日 12_36_18 (1).png`：对局模式，蓝色主色。
- `ChatGPT Image 2026年4月30日 12_36_19 (2).png`：回忆模式，绿色主色。
- `ChatGPT Image 2026年4月30日 12_36_19 (3).png`：复盘模式，紫色主色。

实现时以参考图的首屏视觉为准：白色桌面应用壳、顶部工具栏、三栏工作台、中央大棋盘、左右卡片面板、底部工作区栏。文档中没有明确要求的复杂功能不要额外展开。

核心复现原则：

1. 棋盘始终是视觉中心，占据页面最大、最稳定的区域。
2. 顶部只展示黑白状态、模式分段标签、当前模式动作。
3. 左栏承载当前模式任务，右栏承载辅助信息和分析空态。
4. 整体保持浅色、干净、桌面应用感，不做成 SaaS 大屏或营销页。
5. 三种模式共享布局和组件语言，只通过主色、文案、卡片内容和底部工具差异区分。

## 1. 全局布局与尺寸

以 `1440 x 1000` 左右桌面窗口为基准，页面结构为：

```txt
┌──────────────────────────────────────────────────────────────┐
│ macOS traffic lights / app chrome                            │
├──────────────────────────────────────────────────────────────┤
│ Top Toolbar: 黑白提子状态 | 模式标签 | 当前模式动作           │
├───────────────┬──────────────────────────────┬───────────────┤
│ Left Panels   │          Board Stage          │ Right Panels  │
│ 当前模式任务  │          木纹棋盘              │ 辅助/分析/反馈 │
├───────────────┴──────────────────────────────┴───────────────┤
│ Bottom Workspace Bar: 工作区状态 / 工具 / 缩放视图             │
└──────────────────────────────────────────────────────────────┘
```

推荐尺寸：

- 页面背景：全屏极浅灰白，四周留白约 `16px`。
- 顶部工具栏：高度 `72-84px`，白底，细边框，轻阴影。
- 左栏：`290-330px`，卡片纵向排列。
- 右栏：`290-330px`，卡片纵向排列。
- 中央棋盘区：占剩余主空间，棋盘尽量保持正方形。
- 底部工作区栏：高度 `64-76px`，白底，固定在底部。
- 主体区域间距：左右栏与棋盘之间约 `32-44px`，不要挤压棋盘。

### 1.1 棋盘稳定性约束

棋盘是本产品最核心的视觉锚点。以下约束优先级高于一切侧栏和工具栏需求：

- 切换模式时，棋盘中心点、棋盘尺寸、棋盘坐标位置不得发生明显跳动。
- 左右面板内容变化不能导致棋盘重新布局。
- 底部工具栏展开/收起不能挤压棋盘。
- 棋盘最小可用尺寸不低于 `560px`。
- 当窗口宽度不足时，优先折叠右栏，其次压缩左栏，最后才缩小棋盘。

### 1.2 窄窗口响应式规则

```txt
>= 1280px  三栏全展示
1000-1279  右栏折叠为抽屉，左栏收窄至 260px
< 1000px   左栏也折叠，棋盘居中占满，侧栏变为可唤出抽屉
```

## 2. 顶部工具栏

顶部工具栏是三种模式共用的固定结构。

左侧黑白状态区：

```txt
● 黑棋
  提子 0

○ 白棋
  提子 0
```

要求：

- 黑白棋子使用圆形棋子质感，黑棋深色径向阴影，白棋浅灰立体阴影。
- 黑白状态之间用淡竖线分隔。
- 文案保持小字号、低噪音，不抢中央模式标签。

中间模式分段标签：

```txt
[ 对局模式 ] [ 回忆模式 ] [ 复盘模式 ]
```

要求：

- 使用一个整体圆角分段容器，非激活项白底或透明底。
- 激活项使用模式主色实底：对局蓝、回忆绿、复盘紫。
- 每项可带简洁图标，图标与文字同色。
- 不显示"切换模式""退出本模式""训练模式"。

右侧动作区按模式切换：

- 对局模式：`+ 新对局`、`对局设置`、`认输`。
- 回忆模式：顶部右侧保留"棋局级全局动作"（`+ 新对局`、`对局设置`、`认输`），保持顶部结构一致。当前回忆任务动作（`校对/跳过`、`提示`、`结束回忆`）放在左栏核心任务卡片中。
- 复盘模式：`+ 新建题目`、`复盘设置`、一个网格/列表图标按钮、`认输`。

按钮要求：

- 主按钮高度约 `44px`。
- 普通按钮白底细边框。
- 当前模式主操作使用模式主色文字或主色实底，按参考图处理。
- `认输` 使用红色描边和红色文字，带旗帜图标。在对局/回忆模式中语义为"放弃当前棋局"，在复盘模式中语义为"退出复盘"。

## 3. 中央棋盘舞台

棋盘舞台是每个模式的视觉核心。

棋盘要求：

- 使用浅木纹棋盘，带轻微厚度和阴影。
- 棋盘外框比网格略深，边缘有轻微立体感。
- 网格为细深灰线，星位为小黑点。
- 棋子使用立体阴影，不使用扁平纯色圆。
- 棋盘大小应明显大于任一侧栏卡片组合的单个视觉块。

对局模式棋盘：

- 展示少量已有黑棋，参考图中约 3 枚黑棋。
- 不叠加候选点、编号或复杂分析标记。

回忆模式棋盘：

- 展示回忆落子编号，白棋和黑棋上有 `1`、`2`、`3` 等数字。
- 编号清晰居中，字号不要过大。
- 错误或提示状态优先放在侧栏，棋盘标记保持克制。

复盘模式棋盘：

- 展示少量已有黑棋。
- 默认不显示大量标注，复盘标注工具放在底部工具栏。

## 4. 左右面板通用规范

所有侧栏内容使用白色卡片式 Panel：

- 背景：白色或接近白色。
- 边框：`1px` 极浅灰。
- 圆角：`14-16px`。
- 阴影：轻、柔，不使用重投影。
- 内边距：`18-22px`。
- 卡片间距：`14-16px`。
- 标题字号约 `14px`，正文约 `12-13px`。

面板内容应贴近参考图的密度：

- 左栏卡片可以略大，承载任务说明和主按钮。
- 右栏卡片多为空态、数据摘要或轻量图示。
- 不要把日志、长列表、复杂树结构铺满侧栏。
- 图示空态可使用淡色插图或简洁线形图标，但必须低对比。

### 4.1 右栏卡片展开规则

右栏多处出现"展开"按钮（AI 分析、变化树、快照对比），统一行为：

- 默认展开不改变棋盘尺寸和三栏布局。
- 点击"展开"后，打开右侧抽屉覆盖层。
- 抽屉宽度 `420-520px`，从右侧滑出，覆盖在工作台上方。
- 抽屉不触发三栏重新布局。
- `Esc` 或点击关闭按钮收起抽屉。
- 抽屉内容可展示更详细的分析数据、完整变化树或快照对比视图。

| 状态 | 顶部右侧 | 左栏状态 | 底部栏 | 棋盘 |

## 6. 回忆模式复现规格

主色：绿色 `#169b55` 或接近参考图的高亮绿。

顶部：

```txt
[ 对局模式 ] [ 回忆模式(active) ] [ 复盘模式 ]        [ + 新对局 ] [ 对局设置 ] [ 认输 ]
```

左栏仅保留一个核心任务卡片：

```txt
当前模式

回忆模式
在记忆中回忆下一手

全局回忆任务

      50%

进度      4 / 6
正确      3 手
状态      进行中

上一手需要复盘

[ 校对 / 跳过 ] [ 提示 ] [ 结束回忆 ]
```

要求：

- 环形进度图为主要视觉元素，绿色进度。
- `上一手需要复盘` 使用红色提示。
- `校对 / 跳过` 为蓝色主按钮，`提示` 为普通按钮，`结束回忆` 为红色实底或强警示按钮。
- 不出现阶段选择、范围选择、多层提示等级。

右栏卡片顺序：

1. 回忆提示
2. 结果反馈
3. 轻量变化树

回忆提示卡片：

```txt
回忆提示

下一手：保持回忆
需要时从左侧打开提示。

上一手实际应为 dc。
```

要求：

- 提示结论使用淡黄色提示条。
- 不做复杂提示等级。

结果反馈卡片：

```txt
结果反馈

正确      3
错误      7

总进度    4 / 6
[progress]
```

轻量变化树：

```txt
轻量变化树

竖向节点线
当前节点用蓝色或绿色描边高亮
```

底部工作区栏：

```txt
回忆工作区 | 当前进度 4 / 6 | 等待输入下一手                  [选择] [手型] [-] [+] [全屏]
```

## 8. 模式切换规则

模式之间存在自然的棋局流程，同时也支持用户手动切换：

### 8.1 默认流程

```txt
对局模式 →（对局结束）→ 回忆模式 →（回忆完成）→ 复盘模式 →（可生成题目）
```

- 对局结束后，默认引导进入回忆模式（背谱是基本功）。
- 回忆完成后，引导进入复盘模式。
- 复盘模式可生成题目，题目进入后续训练/复习池。

### 8.2 手动切换规则

- 点击顶部模式标签只切换工作视图，不销毁当前棋谱状态。
- 每个模式保留自己的内部状态，切换回来时恢复到离开时的状态。
- 如果当前有未保存棋局，切换模式前先自动保存；保存失败才弹确认提示。
- 在回忆/复盘模式下点击 `+ 新对局` 等同于从对局模式点击 `+ 新对局`，会弹出确认（当前棋谱是否保存）。
- `认输` 在对局模式下为放弃当前棋局；在回忆模式下为结束回忆；在复盘模式下为退出复盘回到对局模式。

### 8.3 模式切换视觉行为

- 切换模式时，仅左右栏内容和主色变化。
- 棋盘保持不动，无跳动、无缩放、无重新布局。
- 顶部工具栏结构不变，仅右侧动作按钮和模式标签高亮切换。
- 底部工具栏结构不变，仅文案和模式相关工具切换。

## 9. 底部工作区栏

底部栏三种模式共用白色浮层风格。

通用要求：

- 固定在页面底部，左右留白与顶部工具栏一致。
- 高度约 `64-76px`。
- 白底、细边框、轻阴影、圆角。
- 左侧显示当前工作区和状态。
- 右侧显示工具、缩放、全屏或模式相关控制。

对局和回忆模式右侧工具：

```txt
[指针] [手型] [-] [+] [全屏]
```

复盘模式右侧工具：

```txt
标注工具组 + 撤销/重做/清空 + 缩放
```

### 9.1 底部栏动作边界

底部栏只放视图控制、标注工具、缩放、选择/拖拽等操作。

以下动作**不得**放在底部栏：
- 改变棋局流程的动作（认输、结束回忆、新对局）
- 创建新棋局或结束任务的动作
- 生成题目的动作
- 提示、校对等业务主操作

这些动作应放在顶部右侧或左栏卡片内。

## 10. 状态覆盖要求

每个模式至少实现以下六种状态变体，确保不只是"静态正确"的截图：

1. **默认空态** — 首屏参考图状态，无数据。
2. **进行中状态** — 对局第 N 手、回忆进度 N/M、复盘步骤 ②。
3. **成功反馈状态** — 回忆正确、AI 分析完成、题目生成成功。
4. **错误/警告状态** — 引擎连接失败、回忆输入错误、保存失败。
5. **Loading 状态** — 引擎思考中、AI 分析中、保存中。
6. **Disabled 状态** — 未连接引擎时的分析按钮、未选择局面时的标注工具。

## 13. 验收标准

### 13.1 视觉验收 checklist

全局验收：

- [ ] 三种模式首屏分别接近三张参考图，而不是旧版 Sabaki 默认布局。
- [ ] 顶部工具栏包含黑白状态、三段模式标签、右侧动作按钮。
- [ ] 顶部没有"切换模式""退出本模式""训练模式"。
- [ ] 棋盘是页面最大视觉中心，左右栏没有压缩棋盘。
- [ ] 左右面板为白色圆角卡片，边框和阴影克制。
- [ ] 底部工作区栏存在，并按模式展示状态和工具。

对局模式验收：

- [ ] `对局模式` 标签为蓝色激活态。
- [ ] 左栏显示当前模式、当前行棋、对局状态和当前任务。
- [ ] 左栏有 `+ 新对局`、`对局设置`、`认输` 三个操作。
- [ ] 右栏显示局面信息、AI 分析空态、变化树空态。
- [ ] 底部显示"对局工作区 / 当前第 0 手 / 未连接引擎"。

回忆模式验收：

- [ ] `回忆模式` 标签为绿色激活态。
- [ ] 左栏以全局回忆任务为核心，有环形进度 `50%`。
- [ ] 左栏显示进度、正确、状态和 `上一手需要复盘`。
- [ ] 右栏显示回忆提示、结果反馈、轻量变化树。
- [ ] 棋盘显示带编号的回忆落子。
- [ ] 底部显示"回忆工作区 / 当前进度 4 / 6 / 等待输入下一手"。

复盘模式验收：

- [ ] `复盘模式` 标签为紫色激活态。
- [ ] 左栏复盘流程为轻量步骤条，仅当前步骤展开详情。
- [ ] 左栏显示关键点筛选、复盘笔记和 `生成题目`。
- [ ] 右栏显示 AI 分析、局面点评、变化树、快照对比。
- [ ] 底部中央显示标注工具条。
- [ ] 底部右侧显示撤销、重做、清空和缩放。
- [ ] 不默认展示旧规格里的长复盘节点列表。

### 13.2 交互验收 checklist

模式切换：

- [ ] 点击顶部模式标签切换视图，棋盘不跳动。
- [ ] 切换模式后左右栏内容正确更新，主色切换。
- [ ] 切换回来时恢复之前的模式内部状态。
- [ ] 未保存棋局切换时自动保存。

棋盘稳定性：

- [ ] 左右面板内容变化不导致棋盘重新布局。
- [ ] 底部工具栏展开/收起不挤压棋盘。
- [ ] 右栏"展开"打开抽屉覆盖层，不改变三栏布局。

状态覆盖：

- [ ] 每个模式至少覆盖：空态、进行中、成功、错误、loading、disabled。

### 13.3 截图验收

- 为三种模式分别创建固定 route 或 Storybook story。
- 使用相同 mock data 渲染。
- 在 `1440x1000` 下截图。
- 与 reference PNG 进行人工视觉对比。
- 重点检查：
  1. 棋盘是否最大且居中稳定
  2. 左右栏是否低噪音
  3. 顶部模式标签是否清晰
  4. 底部栏是否像工具栏而不是业务区
  5. 三种模式是否只是主色和内容变化，而不是三套完全不同 UI

## 14. 非目标

当前复现阶段不要做：

1. 不要大改棋盘木纹、网格和棋子质感之外的核心棋盘行为。
2. 不要实现完整训练模式页面。
3. 不要加入复杂引擎日志作为默认左栏模块。
4. 不要把回忆模式做成分段训练或多层提示系统。
5. 不要把复盘模式做成纯长列表笔记工具。
6. 不要让 AI 分析、变化树或快照对比比棋盘更抢眼。
7. 不要补充参考图没有表达的重型功能入口。

## 15. 给 Coding Agent 的一句话任务描述

请以 `workbench-ref-pics` 当前三张 PNG 为视觉真源，复现一个浅色、现代、克制的 Sabaki 三栏工作台：顶部为黑白状态、三段模式标签和当前模式动作；中央木纹棋盘始终最大且切换模式时不跳动；左侧展示当前模式任务；右侧展示辅助信息和分析空态（展开时使用右侧抽屉覆盖层）；底部展示工作区状态与工具（仅放视图控制，不放业务主操作）。对局用蓝色，回忆用绿色，复盘用紫色。模式切换只变左右栏内容和主色，不动棋盘。每个模式至少实现空态、进行中、成功、错误、loading 和 disabled 六种状态。
