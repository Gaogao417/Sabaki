# Gabaki / Sabaki 工作台 UI/UX 参考图复现规格

> 目标：以前端实现能够高保真复现 `workbench-ref-pics` 当前三张参考图为准，定义 Sabaki 魔改工作台的布局、视觉层级、模式差异和验收标准。本文优先服务 UI 复现，不再作为完整产品功能蓝图。

## 0. 参考图复现目标

本文的视觉真源是 `docs/design/workbench-ref-pics/` 下当前三张 PNG：

- `ChatGPT Image 2026年4月30日 12_36_18 (1).png`：对局模式，蓝色主色。
- `ChatGPT Image 2026年4月30日 12_36_19 (2).png`：回忆模式，绿色主色。
- `ChatGPT Image 2026年4月30日 12_36_19 (3).png`：复盘模式，紫色主色。

实现时以参考图的首屏视觉为准：白色桌面应用壳、顶部工具栏、三栏工作台、中央大棋盘、左右卡片面板、底部工作区栏。文档中没有明确要求的复杂功能不要额外展开。

核心复现原则：

1. 棋盘始终是视觉中心，占据页面最大、最稳定的区域。
2. 顶部只展示黑白状态、模式分段标签、当前模式动作。
3. 左栏承载当前模式任务，右栏承载辅助信息和分析空态。
4. 整体保持浅色、干净、桌面应用感，不做成 SaaS 大屏或营销页。
5. 三种模式共享布局和组件语言，只通过主色、文案、卡片内容和底部工具差异区分。

## 1. 全局布局与尺寸

以 `1440 x 1000` 左右桌面窗口为基准，页面结构为：

```txt
┌──────────────────────────────────────────────────────────────┐
│ macOS traffic lights / app chrome                            │
├──────────────────────────────────────────────────────────────┤
│ Top Toolbar: 黑白提子状态 | 模式标签 | 当前模式动作           │
├───────────────┬──────────────────────────────┬───────────────┤
│ Left Panels   │          Board Stage          │ Right Panels  │
│ 当前模式任务  │          木纹棋盘              │ 辅助/分析/反馈 │
├───────────────┴──────────────────────────────┴───────────────┤
│ Bottom Workspace Bar: 工作区状态 / 工具 / 缩放视图             │
└──────────────────────────────────────────────────────────────┘
```

推荐尺寸：

- 页面背景：全屏极浅灰白，四周留白约 `16px`。
- 顶部工具栏：高度 `72-84px`，白底，细边框，轻阴影。
- 左栏：`290-330px`，卡片纵向排列。
- 右栏：`290-330px`，卡片纵向排列。
- 中央棋盘区：占剩余主空间，棋盘尽量保持正方形。
- 底部工作区栏：高度 `64-76px`，白底，固定在底部。
- 主体区域间距：左右栏与棋盘之间约 `32-44px`，不要挤压棋盘。

窄窗口下优先压缩左右栏和栏间距，棋盘不得缩小到不可用状态。

## 2. 顶部工具栏

顶部工具栏是三种模式共用的固定结构。

左侧黑白状态区：

```txt
● 黑棋
  提子 0

○ 白棋
  提子 0
```

要求：

- 黑白棋子使用圆形棋子质感，黑棋深色径向阴影，白棋浅灰立体阴影。
- 黑白状态之间用淡竖线分隔。
- 文案保持小字号、低噪音，不抢中央模式标签。

中间模式分段标签：

```txt
[ 对局模式 ] [ 回忆模式 ] [ 复盘模式 ]
```

要求：

- 使用一个整体圆角分段容器，非激活项白底或透明底。
- 激活项使用模式主色实底：对局蓝、回忆绿、复盘紫。
- 每项可带简洁图标，图标与文字同色。
- 不显示“切换模式”“退出本模式”“训练模式”。

右侧动作区按模式切换：

- 对局模式：`+ 新对局`、`对局设置`、`认输`。
- 回忆模式：参考图仍展示 `+ 新对局`、`对局设置`、`认输`，保持顶部结构一致。
- 复盘模式：`+ 新建题目`、`复盘设置`、一个网格/列表图标按钮、`认输`。

按钮要求：

- 主按钮高度约 `44px`。
- 普通按钮白底细边框。
- 当前模式主操作使用模式主色文字或主色实底，按参考图处理。
- `认输` 使用红色描边和红色文字，带旗帜图标。

## 3. 中央棋盘舞台

棋盘舞台是每个模式的视觉核心。

棋盘要求：

- 使用浅木纹棋盘，带轻微厚度和阴影。
- 棋盘外框比网格略深，边缘有轻微立体感。
- 网格为细深灰线，星位为小黑点。
- 棋子使用立体阴影，不使用扁平纯色圆。
- 棋盘大小应明显大于任一侧栏卡片组合的单个视觉块。

对局模式棋盘：

- 展示少量已有黑棋，参考图中约 3 枚黑棋。
- 不叠加候选点、编号或复杂分析标记。

回忆模式棋盘：

- 展示回忆落子编号，白棋和黑棋上有 `1`、`2`、`3` 等数字。
- 编号清晰居中，字号不要过大。
- 错误或提示状态优先放在侧栏，棋盘标记保持克制。

复盘模式棋盘：

- 展示少量已有黑棋。
- 默认不显示大量标注，复盘标注工具放在底部工具栏。

## 4. 左右面板通用规范

所有侧栏内容使用白色卡片式 Panel：

- 背景：白色或接近白色。
- 边框：`1px` 极浅灰。
- 圆角：`14-16px`。
- 阴影：轻、柔，不使用重投影。
- 内边距：`18-22px`。
- 卡片间距：`14-16px`。
- 标题字号约 `14px`，正文约 `12-13px`。

面板内容应贴近参考图的密度：

- 左栏卡片可以略大，承载任务说明和主按钮。
- 右栏卡片多为空态、数据摘要或轻量图示。
- 不要把日志、长列表、复杂树结构铺满侧栏。
- 图示空态可使用淡色插图或简洁线形图标，但必须低对比。

## 5. 对局模式复现规格

主色：蓝色 `#2563ff` 或接近参考图的高亮蓝。

顶部：

```txt
[ 对局模式(active) ] [ 回忆模式 ] [ 复盘模式 ]        [ + 新对局 ] [ 对局设置 ] [ 认输 ]
```

左栏卡片顺序：

1. 当前模式
2. 当前任务

当前模式卡片内容：

```txt
当前模式

对局模式
与电脑或引擎进行对局练习

当前行棋
● 黑棋
您执黑先行

对局状态
● 准备开始
第 0 手
```

当前任务卡片内容：

```txt
当前任务

开始对局
可在下方操作开始新的对局，或调整设置后开始。

下一步操作
[ + 新对局 ]
[ 对局设置 ]
[ 认输 ]
```

右栏卡片顺序：

1. 局面信息
2. AI 分析
3. 变化树

局面信息卡片：

```txt
局面信息

手数        0

```

AI 分析卡片为空态：

```txt
AI 分析

暂无分析数据

```

变化树卡片为空态：


底部工作区栏：

```txt
对局工作区 | 当前第 0 手 | 未连接引擎                         [选择] [手型] [-] [+] [全屏]
```

## 6. 回忆模式复现规格

主色：绿色 `#169b55` 或接近参考图的高亮绿。

顶部：

```txt
[ 对局模式 ] [ 回忆模式(active) ] [ 复盘模式 ]        [ + 新对局 ] [ 对局设置 ] [ 认输 ]
```

左栏仅保留一个核心任务卡片：

```txt
当前模式

回忆模式
在记忆中回忆下一手

全局回忆任务

      50%

进度      4 / 6
正确      3 手
状态      进行中

上一手需要复盘

[ 校对 / 跳过 ] [ 提示 ] [ 结束回忆 ]
```

要求：

- 环形进度图为主要视觉元素，绿色进度。
- `上一手需要复盘` 使用红色提示。
- `校对 / 跳过` 为蓝色主按钮，`提示` 为普通按钮，`结束回忆` 为红色实底或强警示按钮。
- 不出现阶段选择、范围选择、多层提示等级。

右栏卡片顺序：

1. 回忆提示
2. 结果反馈
3. 轻量变化树

回忆提示卡片：

```txt
回忆提示

下一手：保持回忆
需要时从左侧打开提示。

上一手实际应为 dc。
```

要求：

- 提示结论使用淡黄色提示条。
- 不做复杂提示等级。

结果反馈卡片：

```txt
结果反馈

正确      3
错误      7

总进度    4 / 6
[progress]
```

轻量变化树：

```txt
轻量变化树

竖向节点线
当前节点用蓝色或绿色描边高亮
```

底部工作区栏：

```txt
回忆工作区 | 当前进度 4 / 6 | 等待输入下一手                  [选择] [手型] [-] [+] [全屏]
```

## 7. 复盘模式复现规格

主色：紫色 `#7c3fed` 或接近参考图的高亮紫。

顶部：

```txt
[ 对局模式 ] [ 回忆模式 ] [ 复盘模式(active) ]      [ + 新建题目 ] [ 复盘设置 ] [网格] [认输]
```

左栏卡片顺序：

1. 当前模式
2. 复盘流程
3. 关键点筛选
4. 复盘笔记
5. 生成题目按钮

当前模式区：

```txt
当前模式

复盘模式
定位关键局面，分析变化，沉淀笔记
```

复盘流程卡片：

```txt
复盘流程

1 定位关键局面
  选择复盘的关键时刻

2 标注好手与对手
  标记双方关键手法

3 对比参考变化
  分析多种变化与结果

4 沉淀复盘笔记
  记录思考与改进
```

要求：

- 使用紫色圆形步骤编号。
- 步骤之间用紫色虚线连接。
- 不默认展示长节点列表，长列表不是当前参考图首屏目标。

关键点筛选卡片：

```txt
关键点筛选

[ 全部 ] [ 关键点 ] [ 失误 ] [ 备注 ]
使用右侧变化树和局面点评辅助筛选。
```

复盘笔记卡片：

```txt
复盘笔记
[编辑图标]

记录这一手的想法、对局思路与改进方向。
```

底部左栏主按钮：

```txt
[ 生成题目 ]
```

右栏卡片顺序：

1. AI 分析
2. 局面点评
3. 变化树
4. 快照对比

AI 分析卡片为空态：

```txt
AI 分析                         展开

暂无分析数据
选择关键局面后，AI 将在此提供形势判断、推荐手段与变化建议。
```

局面点评卡片：

```txt
局面点评

手数        0
提子        黑 0 / 白 0
综合评价    --
```

变化树卡片：

```txt
变化树                         展开

暂无变化
在对局过程中将自动记录变化，或通过标注生成变化。
```

快照对比卡片：

```txt
快照对比                       展开

捕捉参考局面后可进行快照对比。
[ 添加快照 ]
```

底部工作区栏：

```txt
复盘工作区 | 当前第 0 手 | 关键点 0 · 题目 0

标注工具 [黑白] [X] [△] [□] [○] [线] [箭头] [A] [1]

[撤销] [重做 disabled] [清空] [100%]
```

要求：

- 标注工具条居中，白色浮层按钮组。
- 当前选中工具有浅灰/浅紫背景。
- 撤销、重做、清空、缩放放右侧。

## 8. 底部工作区栏

底部栏三种模式共用白色浮层风格。

通用要求：

- 固定在页面底部，左右留白与顶部工具栏一致。
- 高度约 `64-76px`。
- 白底、细边框、轻阴影、圆角。
- 左侧显示当前工作区和状态。
- 右侧显示工具、缩放、全屏或模式相关控制。

对局和回忆模式右侧工具：

```txt
[指针] [手型] [-] [+] [全屏]
```

复盘模式右侧工具：

```txt
标注工具组 + 撤销/重做/清空 + 缩放
```

不要在底部栏堆放业务主操作；主操作应在顶部右侧或左栏卡片内。

## 10. 验收标准

全局验收：

- [ ] 三种模式首屏分别接近三张参考图，而不是旧版 Sabaki 默认布局。
- [ ] 顶部工具栏包含黑白状态、三段模式标签、右侧动作按钮。
- [ ] 顶部没有“切换模式”“退出本模式”“训练模式”。
- [ ] 棋盘是页面最大视觉中心，左右栏没有压缩棋盘。
- [ ] 左右面板为白色圆角卡片，边框和阴影克制。
- [ ] 底部工作区栏存在，并按模式展示状态和工具。

对局模式验收：

- [ ] `对局模式` 标签为蓝色激活态。
- [ ] 左栏显示当前模式、当前行棋、对局状态和当前任务。
- [ ] 左栏有 `+ 新对局`、`对局设置`、`认输` 三个操作。
- [ ] 右栏显示局面信息、AI 分析空态、变化树空态。
- [ ] 底部显示“对局工作区 / 当前第 0 手 / 未连接引擎”。

回忆模式验收：

- [ ] `回忆模式` 标签为绿色激活态。
- [ ] 左栏以全局回忆任务为核心，有环形进度 `50%`。
- [ ] 左栏显示进度、正确、状态和 `上一手需要复盘`。
- [ ] 右栏显示回忆提示、结果反馈、轻量变化树。
- [ ] 棋盘显示带编号的回忆落子。
- [ ] 底部显示“回忆工作区 / 当前进度 4 / 6 / 等待输入下一手”。

复盘模式验收：

- [ ] `复盘模式` 标签为紫色激活态。
- [ ] 左栏显示四步复盘流程、关键点筛选、复盘笔记和 `生成题目`。
- [ ] 右栏显示 AI 分析、局面点评、变化树、快照对比。
- [ ] 底部中央显示标注工具条。
- [ ] 底部右侧显示撤销、重做、清空和缩放。
- [ ] 不默认展示旧规格里的长复盘节点列表。

## 11. 非目标

当前复现阶段不要做：

1. 不要大改棋盘木纹、网格和棋子质感之外的核心棋盘行为。
2. 不要实现完整训练模式页面。
3. 不要加入复杂引擎日志作为默认左栏模块。
4. 不要把回忆模式做成分段训练或多层提示系统。
5. 不要把复盘模式做成纯长列表笔记工具。
6. 不要让 AI 分析、变化树或快照对比比棋盘更抢眼。
7. 不要补充参考图没有表达的重型功能入口。

## 12. 给 Coding Agent 的一句话任务描述

请以 `workbench-ref-pics` 当前三张 PNG 为视觉真源，复现一个浅色、现代、克制的 Sabaki 三栏工作台：顶部为黑白状态、三段模式标签和当前模式动作；中央木纹棋盘始终最大；左侧展示当前模式任务；右侧展示辅助信息和分析空态；底部展示工作区状态与工具。对局用蓝色，回忆用绿色，复盘用紫色，整体必须接近参考图首屏。
